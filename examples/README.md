# Examples

End-to-end worked runs: requirement → oracle → test code → execution → report.

These are real (or realistic) scenarios you can read, learn from, and adapt.

## What's Here

- **sample-prd/** — A real PRD for a feature. Walk through it to see:
  - How to extract an oracle from a requirement
  - How to author tests against that oracle
  - How to run them and interpret results
  - How to handle failures (heal stage)

This example is designed to be followed linearly — you can read it end-to-end and see the full loop.

## How to Use

1. **Read as a learning tool** — understand the flow without running anything
2. **Follow it step-by-step** — use the prompts in `/prompts/[stage]` to reproduce the work
3. **Adapt to your own requirement** — when you have your own PRD, follow the same pattern
